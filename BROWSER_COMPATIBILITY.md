# 瀏覽器兼容性測試清單

## 支援的瀏覽器

### 桌面瀏覽器
- ✅ **Chrome 80+** (推薦)
- ✅ **Firefox 80+**
- ✅ **Safari 14+** (macOS)
- ✅ **Edge 80+** (Chromium)

### 行動瀏覽器
- ✅ **iOS Safari 14+**
- ✅ **Chrome Android 80+**
- ✅ **Samsung Internet 14+**

## 技術要求

### 必要功能
1. **WebGL 支援** - Mapbox GL JS 需要 WebGL 1.0 或 2.0
2. **ES2020+ JavaScript** - 現代 JavaScript 語法
3. **CSS Grid/Flexbox** - 佈局系統
4. **CSS backdrop-filter** - 毛玻璃效果（有降級處理）

### 不支援的瀏覽器
- ❌ Internet Explorer 11 及更早版本
- ❌ 不支援 WebGL 的瀏覽器
- ❌ 過舊的行動瀏覽器（iOS < 14, Android < 8）

## 測試檢查項目

### 基本功能
- [ ] 地圖載入正常
- [ ] 地圖縮放/平移流暢
- [ ] 軍演範圍圖層顯示
- [ ] 參考圖層顯示（ADIZ、海峽中線、領海基線、領海）
- [ ] 語言切換功能正常
- [ ] 圖層控制面板正常運作

### 互動功能
- [ ] 勾選/取消軍演範圍
- [ ] 地圖控制按鈕（縮放、旋轉）
- [ ] 響應式設計（手機/平板/桌面）

### 效能
- [ ] 首次載入時間 < 3秒
- [ ] 圖層切換無延遲
- [ ] 無記憶體洩漏（長時間使用）

## 測試方法

### 手動測試
1. 在不同瀏覽器中打開應用
2. 檢查控制台是否有錯誤
3. 測試所有互動功能
4. 檢查響應式設計

### 自動化測試（可選）
使用 BrowserStack 或類似服務進行跨瀏覽器測試

## 已知問題

### Safari
- backdrop-filter 在較舊版本可能不支援（已有降級處理）

### Firefox
- 某些 CSS 動畫可能需要 `-moz-` 前綴（已處理）

## 降級處理

1. **backdrop-filter**: 如果瀏覽器不支援，會使用半透明背景
2. **WebGL**: 如果瀏覽器不支援，會顯示錯誤訊息
3. **ES2020**: Vite 會自動轉譯為相容的 JavaScript
